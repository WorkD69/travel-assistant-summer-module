# Unified Travel Assistant Backend Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build, test, deploy, and connect one PostgreSQL-backed API for the existing Vercel frontend and VPS Telegram bot without touching the VPN.

**Architecture:** A dedicated Express Vercel project serves both cookie-authenticated site routes and service-authenticated bot routes. A free Neon PostgreSQL database is the source of truth; the frontend reaches it through a same-origin Vercel rewrite and the Telegram bot reaches the backend HTTPS URL directly.

**Tech Stack:** Node.js, Express, Prisma/PostgreSQL, Vitest, Supertest, Vercel, Neon, static HTML/CSS/JavaScript, Python `HttpTravelApiClient`, Aiogram systemd service.

---

## File map

The backend is split by responsibility instead of copying the large ZIP route files:

- `backend/src/config.js` — environment parsing and startup validation.
- `backend/src/db.js` — singleton Prisma client and readiness query.
- `backend/src/errors.js` — typed errors and OpenAPI-compatible envelope.
- `backend/src/app.js` — Express composition only.
- `backend/src/security/` — cookie, Origin/CORS, service auth, hashing, rate limits.
- `backend/src/access/` — trip membership and role/object authorization.
- `backend/src/routes/site/` — browser API routes.
- `backend/src/routes/bot/` — immutable Telegram OpenAPI routes.
- `backend/src/services/` — SOS, Plan B, outbox, document tokens, context filtering.
- `backend/prisma/` — PostgreSQL schema, migrations, deterministic safe seed.
- `backend/tests/` — unit, database integration, OpenAPI and security tests.
- `frontend/assets/js/api-client.js` — same-origin browser client without demo login.
- `frontend/assets/js/*-sync.js` — narrowly scoped UI adapters.
- `frontend/vercel.json` — `/api/*` production rewrite.
- `telegram-bot/` — handlers remain unchanged; only tests/docs or client fixes required by a proven contract mismatch may change.
