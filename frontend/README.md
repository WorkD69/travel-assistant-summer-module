# Тревел-помощник — финальная объединённая сборка

Единый frontend-проект, собранный из четырёх веток:
основа (workspace поездки), trip-pages (Главная, История, Мастер),
core-flow (Мониторинг, Сообщения, SOS/Plan B), account-pages (вход, профиль).

## Как запустить

1. Распакуйте архив `travel-assistant-final-merged.zip`.
2. Откройте папку `travel-assistant-final`.
3. Запустите локальный сервер:
   - **Windows:** двойной клик по `START_PREVIEW.bat`;
   - **macOS / Linux:** в терминале `./start-preview.sh` (или `bash start-preview.sh`).
4. В браузере откроется `http://localhost:8000/index.html`.

Запуск двойным кликом по HTML-файлу не поддерживается как основной способ —
нужен локальный HTTP-сервер (его и запускают скрипты выше). Требуется установленный Python 3.

## Аккаунты

Production-вход и регистрация работают через backend и защищённую HttpOnly cookie.
Демонстрационные пароли в frontend не хранятся. Учётные данные для safe seed
задаются отдельно через переменные окружения при развёртывании backend.

## Основные экраны

- `index.html` — единая точка входа (перенаправляет на вход или Главную);
- `login.html`, `register.html`, `password-recovery.html`, `invitation.html` — аккаунт;
- `home.html` — Главная: приглашения, активные поездки и черновики;
- `history.html` — история завершённых поездок;
- `trip-wizard.html` — мастер создания/редактирования поездки (8 шагов);
- `trip-overview.html?tripId=<id>` — рабочее пространство поездки
  (Обзор, Маршрут, Документы, Участники, Мониторинг, Сообщения, Настройки);
- `profile.html` — профиль: аккаунт, Telegram, почта, уведомления, оформление.

## Development-панели

Служебные панели скрыты в production-режиме. Чтобы открыть их,
добавьте к адресу страницы параметр `?env=development`, например:
`http://localhost:8000/trip-overview.html?tripId=trip-turkey-2026&env=development`.

Технические preview-страницы веток сохранены в каталоге `dev-preview/`
и не участвуют в пользовательской навигации.

## Что является демонстрацией

- Все данные (аккаунты, поездки, документы, сообщения) — демонстрационные и хранятся в `localStorage` браузера.
- Вход/регистрация/восстановление пароля — frontend-демо, не production-безопасность.
- Подключение Telegram использует production deep link и backend; подключение почты пока остаётся демонстрационным.
- Отправка сообщений и SOS — безопасная имитация без реальных внешних действий.
- Plan B не выполняет реальных покупок, отмен или переоформлений.
