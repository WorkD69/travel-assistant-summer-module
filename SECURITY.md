# Политика безопасности

## Секреты

- Никогда не коммитьте `.env` и его резервные копии.
- Не публикуйте Telegram Bot Token, Groq API Key, backend service token,
  пароли, приватные ключи и VPN-конфигурации.
- Groq и Telegram-секреты не должны попадать во frontend.
- Backend service token никогда не должен передаваться браузеру.
- В `.env.example` допустимы только пустые значения и безопасные примеры.

Локальными и исключёнными из Git считаются `.env`, `.venv/`, `bot_state.db`,
`bot.pid`, `logs/`, кэши Python и тестов, локальные SQLite/DB-файлы и резервные
копии.

Browser-сессия хранится только в cookie `travel_session` с флагами HttpOnly,
Secure в production и SameSite=Lax. Browser mutations проверяют точный Origin.
Bot API требует отдельный Bearer service token; роль всегда определяется по
PostgreSQL membership. Link/download tokens сохраняются только как SHA-256 hash.

Загруженные документы ограничены 5 МБ и allowlist MIME. Backend не выполняет
тяжёлый OCR синхронно: PDF и изображения переводятся в manual review. Текстовые
файлы извлекаются с ограничением размера.

## Если секрет раскрыт

1. Немедленно отзовите или перевыпустите ключ у соответствующего провайдера.
2. Обновите секрет только в локальном `.env` или защищённом хранилище среды.
3. Удалите секрет из рабочего дерева и всей опубликованной Git-истории.
4. Выполните повторный secret scan до следующего коммита или публикации.
5. Проверьте журналы доступа и связанные учётные записи на злоупотребления.

Сообщения об уязвимостях следует передавать владельцу проекта приватно, без
публикации рабочего exploit или реальных секретов.
