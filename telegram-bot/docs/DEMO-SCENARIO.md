# Демо-сценарий защиты

Перед показом: `BOT_ENV=development`, `BOT_DATA_MODE=mock`, `BOT_UPDATE_MODE=polling`, `AI_PROVIDER=mock`; запустите `CHECK_BOT.bat`, затем `START_BOT.bat`.

## Два Telegram-пользователя

- Артём: `https://t.me/<bot_username>?start=link_demo-artem`
- Анна: `https://t.me/<bot_username>?start=link_demo-anna`

Откройте ссылки с двух Telegram-аккаунтов. Token одноразовый в текущем процессе.

## Организатор Артём

1. Привяжите Артёма.
2. `/trips` → «Отпуск в Турции» → выбрать активной.
3. `/today` или `/next`.
4. `/documents` и скачайте вымышленный PDF.
5. `/demo` → задержка; дождитесь уведомления.
6. После SOS Анны получите уведомление.
7. «Открыть SOS на сайте» откроет workspace поездки; вкладку выберите вручную.

## Участник Анна

1. Привяжите Анну.
2. Выберите «Отпуск в Турции».
3. Проверьте `/today`, `/next`, `/messages` и опубликованный Plan B.
4. `/documents`: доступен общий/собственный, закрытый документ отсутствует.
5. `/sos` → сегмент → категория → описание → подтверждение.
6. `/mysos` показывает собственный статус.
7. `/assistant` задайте вопрос по трансферу или Plan B.

## Один Telegram-пользователь последовательно

Используйте `link_demo-artem`, покажите сценарий, затем `/unlink`. Для Анны нужен ещё не использованный `link_demo-anna`. Обратно привязать уже использованный token без перезапуска mock dataset нельзя - это демонстрация одноразовости.
