# Groq и AI-помощник

## Опциональный proxy только для Groq

`GROQ_PROXY_URL` по умолчанию пуст. Если активный VPN предоставляет локальный
HTTP-, HTTPS- или SOCKS5-endpoint, его адрес указывают только в локальном `.env`.
Реальный адрес и учётные данные proxy нельзя добавлять в `.env.example`.

Явный `GROQ_PROXY_URL` имеет приоритет над `HTTP_PROXY`, `HTTPS_PROXY` и
`ALL_PROXY`. При пустом значении Groq-клиент учитывает стандартные переменные
окружения через `trust_env=True`. URL и учётные данные proxy, API-ключи, полный
prompt и тела ответов в логи не записываются.

Groq подключается только внутри Telegram-бота через `AIProvider`. Handlers и
`/assistant` не знают URL, ключ или формат ответа провайдера.

## Локальная конфигурация

В `.env` задаются `AI_PROVIDER=groq`, Groq base URL, основная и резервная
модели, timeout/retry/лимиты и `AI_FALLBACK_TO_MOCK=true`. Секретная переменная
`GROQ_API_KEY` намеренно отсутствует в `.env.example`: добавьте её только в
локальный `.env` и никогда не отправляйте в чат.

Если ключ был опубликован, немедленно отзовите его в Groq Console и создайте
новый. Опубликованный ключ нельзя повторно использовать даже после удаления
сообщения.

## Последовательность

Актуальная цепочка моделей:

```text
llama-3.3-70b-versatile -> openai/gpt-oss-20b -> MockAIProvider
```

Обе Groq-модели используют один локальный `GROQ_API_KEY`; отдельный ключ для резервной модели не нужен.

Бот сначала обращается к основной модели. После ограниченного числа повторов
при временной ошибке он последовательно пробует резервную модель. Ошибки ключа
или доступа не повторяются. Если Groq полностью недоступен, ответ явно
помечается как полученный от `MockAIProvider`.

## Безопасность

Перед AI очищаются текущий вопрос, история и разрешённый контекст поездки.
Полный prompt, ответы API, Authorization header и ключ не логируются. В логах
допустимы только модель, длительность, HTTP-статус, число сообщений, размер
контекста, номер попытки и тип ошибки.

## Проверка

После offline-тестов внесите новый ключ локально, выполните один минимальный
запрос, перезапустите только процесс из `bot.pid` и задайте `/assistant` вопрос
по активной поездке. `BOT_DATA_MODE` при этом остаётся `mock`.
