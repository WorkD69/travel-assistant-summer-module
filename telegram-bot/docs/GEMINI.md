# Gemini и AI-помощник

## Mock (по умолчанию)

```dotenv
AI_PROVIDER=mock
GEMINI_ENABLED=false
```

Ключ не нужен; ответы помечены как demo.

## Gemini

```dotenv
AI_PROVIDER=gemini
GEMINI_ENABLED=true
GEMINI_API_KEY=секрет
GEMINI_MODEL=модель_доступная_вашему_ключу
```

Используется официальный `google-genai`. Модель задаётся только окружением. Без ключа/модели factory безопасно использует MockAIProvider.

## Безопасность

В prompt входит минимальный backend-фильтрованный контекст: разрешённые события, названия доступных документов, опубликованные сообщения/Plan B и собственные SOS. Email, телефон, паспорт, карта, длинные tokens и номера документов маскируются. Текущий вопрос сохраняется только после sanitation; перед каждым запросом заново очищается вся история. Полный prompt не логируется.

## Ошибки

Timeout, 429/quota, unavailable SDK/model/network и invalid response превращаются в типизированные `AIProviderError`. Глобальный middleware показывает безопасное сообщение без traceback; поездки, документы и SOS продолжают работать.

## Проверка

1. Сначала проверьте `AI_PROVIDER=mock` командой `/assistant`.
2. Для Gemini внесите значения в `.env`, перезапустите бот и задайте вопрос по поездке.
3. Не помещайте ключ в ZIP. Реальный Gemini-вызов без пользовательского ключа не заявлен как проверенный.
